The Story So Far

My engineering career started with a simple frustration: watching teams burn hours on things machines should do. At Impactsure that meant building OCR pipelines to auto-process KYC documents and ML classification systems that ran without human involvement — work that quietly freed analysts from repetitive, error-prone review cycles.

At GrayQuest I stepped into senior ownership earlier than my title suggested. I rebuilt the API ingestion path so bulk work happened asynchronously, stabilised a Kafka pipeline that was losing messages under payment traffic load, and built an observability layer that transformed our on-call rotations from reactive firefighting into structured incident management.

What drives me is the compounding effect of well-built systems — where improving reliability in one layer makes the whole product more stable. I'm ready to bring that thinking to a larger, more complex product at SDE-2 or Senior level.

Projects

Problems Solved

01 · LATENCY
Async Ingestion Pipeline
Partners were hitting timeouts during bulk CSV uploads because every row was being processed synchronously on the API thread. Moved all record processing into Celery workers backed by Redis, so the API returns an acknowledgement immediately while work happens in the background.
Endpoint p99 latency: 3 s → 1 s. Zero partner-facing timeouts.
CeleryRedisFastAPIPython
02 · RELIABILITY
Kafka Pipeline Stabilisation
Consumer groups were restarting several times daily during peak payment traffic. Root cause: over-committed consumer threads and no handling for malformed events. Rebalanced partition assignments, introduced Dead Letter Queues, and set Grafana alerts on lag thresholds before they cascade into restarts.
Consumer restarts: multiple/day → near zero.
KafkaDLQPrometheusGrafana
03 · SCALE
Config-Driven Tenant Engine
Every new institution partner needed a developer to write, review, and deploy schema changes before going live. Moved institution-specific admission rules into a config store so that onboarding is a non-engineering operation — product team fills a form, partner is live the same day.
Onboarding time: 5 days → 1 day. Zero per-partner migrations.
PythonFastAPIMySQL
04 · AUTOMATION
Airflow ETL Platform
The data team ran daily reconciliation scripts manually each morning — a fragile, blocking routine prone to silent failures. Replaced everything with scheduled Airflow DAGs: pull transactional data, apply transformations, validate, and push clean datasets to S3. Failures alert; silence means success.
100% of manual daily jobs eliminated. Data pipeline now self-managing.
AirflowPandasAWS S3Python
05 · AI / OCR
Document Intelligence Pipeline
KYC verification required analysts to manually read scanned letters and extract fields. Built a preprocessing → OCR → structured extraction → confidence scoring pipeline. Low-confidence extractions get flagged for human review; high-confidence ones flow through automatically.
Extraction accuracy: 40% → 90%. Human review load cut dramatically.
OCRDjangoQ ClusterPython
